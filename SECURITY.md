# Security Policy

## Reporting A Vulnerability

If you discover a security issue in Kaze, do not disclose it publicly.

Please report it privately to:
- See [CONTACT.md](CONTACT.md)

## What To Include

Please include:

- a short description of the issue
- affected feature or module
- reproduction steps
- impact assessment if known
- screenshots, logs, or proof of concept if relevant

## Response Goal

Kaze is a private business project, so response times may vary by project stage, but valid reports will be reviewed as quickly as possible.

## Scope

Security concerns may include:

- authentication or authorization flaws
- insecure data exposure
- privilege escalation
- notification abuse
- insecure local storage
- backend/API vulnerabilities
- map/access control bypass

## Public Disclosure

Do not publish vulnerabilities, exploit details, or sensitive internal project information without written authorization.
