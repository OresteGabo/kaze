# UI Architecture

## Principles

Kaze should avoid generic stock Material layouts when a more intentional hospitality design is needed.

Current UI goals:
- compact headers
- custom segmented controls instead of default tabs where useful
- a distinctive bottom navigation dock
- full-page map experience
- premium access-pass presentation

## Main Areas

### Stay

Purpose:
- show identity and confirmed itinerary
- show the digital access pass
- keep core travel/hospitality information in one place

### Requests

Purpose:
- group things the visitor wants to ask for
- late checkout
- service requests
- concierge assistance

### Suggestions

Purpose:
- show recommendations that are not yet part of the user’s confirmed experience

### Map

Purpose:
- give venue-scale spatial understanding
- support zoom and floor selection
- visualize restricted and hidden zones

## Current Custom Components

Examples already in the prototype:
- custom bottom bar
- custom segmented stay tabs
- signature access pass
- full-page zoomable map

## Next UI Opportunities

- redesign the events day selector
