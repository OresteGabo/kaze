# Event Platform Vision

## One-Sentence Vision

Kaze helps events feel organized, premium, and easy to navigate for guests, organizers, venues, and service partners.

## Core Positioning

Kaze should be understood as:
- an event operating system
- an access and pass product
- an event-service commerce layer

Kaze should not be positioned as:
- a generic hotel utility app
- a room-service app
- a broad hospitality super-app

## Product Center

The center of the app should be:
- invitations
- RSVP
- pass and check-in
- schedule
- venue/place guidance
- event-linked services

Everything else should either support that center or move out of the main product story.

## The Three Main Sides

### 1. Guests and attendees

They need:
- a clean invitation path
- a clear pass
- easy schedule visibility
- simple venue direction
- relevant add-on services

### 2. Organizers and hosts

They need:
- guest management
- RSVP visibility
- pass and access control
- simple service coordination
- less operational chaos

### 3. Venues and partners

They need:
- event visibility
- smoother guest flow
- service upsell opportunities
- local payment support
- direct relationships instead of fragmented manual coordination

## Revenue Logic

Kaze should make money from:
- organizer software fees
- venue partnerships
- service commissions
- featured provider placement
- packaged event upsells

Then later, Kaze can own the best-performing service categories directly.

Examples:
- Kaze Studio for photo and video
- Kaze Transport
- Kaze Print and Branding
- Kaze Event Styling

## Product Rule For Future Decisions

When deciding whether a feature belongs in Kaze, ask:

Does this help an event guest, organizer, venue, or event-linked service provider move through the event journey better?

If the answer is no, it probably belongs:
- in a secondary module
- in a partner integration
- in a future separate product

## UX Rule

If users need to read too much to understand what a screen is doing, the UI is weak.

Kaze should prefer:
- visual guidance
- strong hierarchy
- event-first language
- fewer but sharper surfaces

## Final Direction

Kaze wins by being focused.

Not by doing everything around hospitality, but by doing the event journey exceptionally well and turning that journey into a platform for access, commerce, and premium services.
